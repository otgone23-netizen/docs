## Git Стратеги

Бид `Gitflow` эсвэл үүнтэй төстэй салбарлах стратегийг баримталдаг.

*   `main` - Production орчин.
*   `develop` - Хөгжүүлэлтийн үндсэн салаа.
*   `feature/task-name` - Шинэ боломж нэмэх үед.
*   `fix/issue-name` - Алдаа засах үед.

### Commit хийх стандарт
Commit message нь тодорхой, ойлгомжтой байх ёстой. Жишээ:
`feat: add login functionality`, `fix: resolve crash on sidebar`

## Code Review

Бүх Pull Request (PR) нь дор хаяж нэг ахлах хөгжүүлэгчийн зөвшөөрөл авсан байх ёстой. PR хийхдээ:
1. Юу өөрчилсөн талаар товч тайлбар бичих.
2. Тест хийсэн үр дүнгээ хавсаргах.
3. Reviewer-т мэдэгдэх.

## Харилцаа холбоо

*   **Daily Standup:** Өглөө бүр 09:30-д хийсэн ажил болон тулгарсан асуудлаа ярилцана.
*   **Task Management:** Jira эсвэл Trello ашиглан ажлаа төлөвлөнө.
*   **Documenting:** Бүх шинэ функцүүд баримтжуулалттай байх ёстой.
